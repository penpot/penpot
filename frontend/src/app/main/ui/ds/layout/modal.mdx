# Modal

Modals are overlays that require the user's attention, 
displaying critical information or actions while blocking interaction with the rest of the page.

## Variants

**Default**, a modal with header, content, and footer.
<Canvas of={Modal.Default} />

**Without title**, a modal with no header or footer.
<Canvas of={Modal.WithoutTitle} />

**Small**, used for confirmations and simple actions.
<Canvas of={Modal.Small} />

**Large**, used for forms or content-heavy dialogs.
<Canvas of={Modal.Large} />

**Non dismissable**, the user must interact with the modal to close it.
<Canvas of={Modal.NonDismissable} />

## Technical notes

### Composition

The Modal provides three named slots: `header`, `content`, `footer`.
Use the CLJS sub-components or pass custom content to each slot:

```clojure
;; Using the :heading shortcut (string → renders heading* + close button)
[:> modal* {:is-open open?
            :on-open-change #(reset! open* %)
            :heading "Dialog Title"}
  [:> modal-content* {}
    [:> :p {} "Content"]]
  [:> modal-footer* {}
    [:> button* {:variant "primary"} "Save"]]]
```

```clojure
;; Full control via explicit slots
[:> modal* {:is-open open?
            :on-open-change #(reset! open* %)
            :header [:> modal-header* {:title "Dialog Title"}]
            :content [:> modal-content* {} [:> :p {} "Content"]]
            :footer [:> modal-footer* {}
                      [:> button* {:variant "secondary"} "Cancel"]
                      [:> button* {:variant "primary"} "Save"]]}]
```

### Slot components

| Component | Description |
|-----------|-------------|
| `modal-header*` | Renders `heading*` + `ModalCloseButton` in a flex container |
| `modal-content*` | Scrollable body container |
| `modal-footer*` | Actions container with gap and flex-end alignment |
| `ModalCloseButton` | Button that closes the modal via context (available as `ui/ModalCloseButton`) |

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `is-open` | boolean | — | Controls modal visibility |
| `on-open-change` | fn | — | Called when modal opens/closes |
| `heading` | string \| element | — | Shortcut for header slot (string → heading* component with close button) |
| `header` | element | — | Custom header content |
| `content` | element | — | Custom content (defaults to children wrapped in modal-content*) |
| `footer` | element | — | Custom footer content |
| `trigger` | element | — | Element that opens the modal |
| `is-dismissable` | boolean | `true` | Close on backdrop click or Escape |
| `size` | `"small" \| "medium" \| "large"` | `"medium"` | Modal width |

### Accessibility

The modal uses `react-aria-components` under the hood, providing:
- Focus trapping within the modal
- Escape key to dismiss
- ARIA `dialog` role and `aria-modal`
- Focus restoration on close
- Backdrop click dismiss (configurable)

## Usage guidelines (design)

### Where to use

When you need to interrupt the user's current task with critical information,
a required decision, or a focused workflow that must be completed before returning
to the main interface.

### When to use

- Confirmations before destructive actions
- Forms that require immediate completion
- Displaying critical information
- Wizards or multi-step flows within a single context

### When not to use

- For non-critical information that can be dismissed (use a toast or notification instead)
- For complex navigation (use a new page or a panel)
- When the content is too long (consider a slide-over or dedicated page)
